Full documentation can be found at \href{https://config.thephpleague.com/}{\texttt{ config.\+thephpleague.\+com}}.\hypertarget{md_vendor_2league_2config_2_r_e_a_d_m_e_autotoc_md1070}{}\doxysection{\texorpdfstring{💭 Philosophy}{💭 Philosophy}}\label{md_vendor_2league_2config_2_r_e_a_d_m_e_autotoc_md1070}
This library aims to provide a {\bfseries{simple yet opinionated}} approach to configuration with the following goals\+:


\begin{DoxyItemize}
\item The configuration should operate on {\bfseries{arrays with nested values}} which are easily accessible
\item The configuration structure should be {\bfseries{defined with strict schemas}} defining the overall structure, allowed types, and allowed values
\item Schemas should be defined using a {\bfseries{simple, fluent interface}}
\item You should be able to {\bfseries{add and combine schemas but never modify existing ones}}
\item Both the configuration values and the schema should be {\bfseries{defined and managed with PHP code}}
\item Schemas should be {\bfseries{immutable}}; they should never change once they are set
\item Configuration values should never define or influence the schemas
\end{DoxyItemize}

As a result, this library will likely {\bfseries{never}} support features like\+:


\begin{DoxyItemize}
\item Loading and/or exporting configuration values or schemas using YAML, XML, or other files
\item Parsing configuration values from a command line or other user interface
\item Dynamically changing the schema, allowed values, or default values based on other configuration values
\end{DoxyItemize}

If you need that functionality you should check out other libraries like\+:


\begin{DoxyItemize}
\item \href{https://symfony.com/doc/current/components/config.html}{\texttt{ symfony/config}}
\item \href{https://symfony.com/doc/current/components/options_resolver.html}{\texttt{ symfony/options-\/resolver}}
\item \href{https://github.com/hassankhan/config}{\texttt{ hassankhan/config}}
\item \href{https://github.com/consolidation/config}{\texttt{ consolidation/config}}
\item \href{https://docs.laminas.dev/laminas-config/}{\texttt{ laminas/laminas-\/config}}
\end{DoxyItemize}\hypertarget{md_vendor_2league_2config_2_r_e_a_d_m_e_autotoc_md1071}{}\doxysection{\texorpdfstring{🏷️ Versioning}{🏷️ Versioning}}\label{md_vendor_2league_2config_2_r_e_a_d_m_e_autotoc_md1071}

When a new {\bfseries{major}} version is released (e.\+g. {\ttfamily 1.\+1} -\/\texorpdfstring{$>$}{>} {\ttfamily 2.\+0}), the previous one ({\ttfamily 1.\+1}) will receive critical bug fixes for {\itshape at least} 3 months and security updates for 6 months after that new release comes out.
