 \hypertarget{md_vendor_2phar-io_2version_2_r_e_a_d_m_e_autotoc_md1363}{}\doxysection{\texorpdfstring{Version constraints}{Version constraints}}\label{md_vendor_2phar-io_2version_2_r_e_a_d_m_e_autotoc_md1363}
A Version constraint describes a range of versions or a discrete version number. The format of version numbers follows the schema of \href{http://semver.org}{\texttt{ semantic versioning}}\+: {\ttfamily \texorpdfstring{$<$}{<}major\texorpdfstring{$>$}{>}.\texorpdfstring{$<$}{<}minor\texorpdfstring{$>$}{>}.\texorpdfstring{$<$}{<}patch\texorpdfstring{$>$}{>}}. A constraint might contain an operator that describes the range.

Beside the typical mathematical operators like {\ttfamily \texorpdfstring{$<$}{<}=}, {\ttfamily \texorpdfstring{$>$}{>}=}, there are two special operators\+:

{\itshape Caret operator}\+: {\ttfamily \texorpdfstring{$^\wedge$}{\string^}1.0} can be written as {\ttfamily \texorpdfstring{$>$}{>}=1.\+0.\+0 \texorpdfstring{$<$}{<}2.\+0.\+0} and read as »every Version within major version {\ttfamily 1}«.

{\itshape Tilde operator}\+: {\ttfamily \texorpdfstring{$\sim$}{\string~}1.0.\+0} can be written as {\ttfamily \texorpdfstring{$>$}{>}=1.\+0.\+0 \texorpdfstring{$<$}{<}1.\+1.\+0} and read as »every version within minor version {\ttfamily 1.\+1}. The behavior of tilde operator depends on whether a patch level version is provided or not. If no patch level is provided, tilde operator behaves like the caret operator\+: {\ttfamily \texorpdfstring{$\sim$}{\string~}1.0} is identical to {\ttfamily \texorpdfstring{$^\wedge$}{\string^}1.0}.\hypertarget{md_vendor_2phar-io_2version_2_r_e_a_d_m_e_autotoc_md1364}{}\doxysection{\texorpdfstring{Usage examples}{Usage examples}}\label{md_vendor_2phar-io_2version_2_r_e_a_d_m_e_autotoc_md1364}
