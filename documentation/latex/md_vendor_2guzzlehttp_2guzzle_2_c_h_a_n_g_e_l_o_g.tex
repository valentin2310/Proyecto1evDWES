The breaking changes in this release are relatively minor. The biggest thing to look out for is that request and response objects no longer implement fluent interfaces.


\begin{DoxyItemize}
\item Removed the fluent interfaces (i.\+e., {\ttfamily return \$this}) from requests, responses, {\ttfamily \doxylink{namespace_guzzle_http}{Guzzle\+Http}\textbackslash{}Collection}, {\ttfamily \doxylink{namespace_guzzle_http}{Guzzle\+Http}\textbackslash{}Url}, {\ttfamily \doxylink{namespace_guzzle_http}{Guzzle\+Http}\textbackslash{}Query}, {\ttfamily \doxylink{namespace_guzzle_http}{Guzzle\+Http}\textbackslash{}Post\textbackslash{}Post\+Body}, and {\ttfamily \doxylink{namespace_guzzle_http}{Guzzle\+Http}\textbackslash{}Cookie\textbackslash{}Set\+Cookie}. This blog post provides a good outline of why I did this\+: \href{https://ocramius.github.io/blog/fluent-interfaces-are-evil/}{\texttt{ https\+://ocramius.\+github.\+io/blog/fluent-\/interfaces-\/are-\/evil/}}. This also makes the Guzzle message interfaces compatible with the current PSR-\/7 message proposal.
\item Removed "{}functions.\+php"{}, so that Guzzle is truly PSR-\/4 compliant. Except for the HTTP request functions from \doxylink{function_8php}{function.\+php}, these functions are now implemented in {\ttfamily \doxylink{namespace_guzzle_http}{Guzzle\+Http}\textbackslash{}Utils} using camel\+Case. {\ttfamily \doxylink{namespace_guzzle_http}{Guzzle\+Http}\textbackslash{}json\+\_\+decode} moved to {\ttfamily \doxylink{namespace_guzzle_http}{Guzzle\+Http}\textbackslash{}Utils\+::json\+Decode}. {\ttfamily \doxylink{namespace_guzzle_http}{Guzzle\+Http}\textbackslash{}get\+\_\+path} moved to {\ttfamily \doxylink{namespace_guzzle_http}{Guzzle\+Http}\textbackslash{}Utils\+::get\+Path}. {\ttfamily \doxylink{namespace_guzzle_http}{Guzzle\+Http}\textbackslash{}set\+\_\+path} moved to {\ttfamily \doxylink{namespace_guzzle_http}{Guzzle\+Http}\textbackslash{}Utils\+::set\+Path}. {\ttfamily \doxylink{namespace_guzzle_http}{Guzzle\+Http}\textbackslash{}batch} should now be {\ttfamily \doxylink{namespace_guzzle_http}{Guzzle\+Http}\textbackslash{}Pool\+::batch}, which returns an {\ttfamily object\+Storage}. Using functions.\+php caused problems for many users\+: they aren\textquotesingle{}t PSR-\/4 compliant, require an explicit include, and needed an if-\/guard to ensure that the functions are not declared multiple times.
\item Rewrote adapter layer.
\begin{DoxyItemize}
\item Removing all classes from {\ttfamily \doxylink{namespace_guzzle_http}{Guzzle\+Http}\textbackslash{}Adapter}, these are now implemented as callables that are stored in {\ttfamily \doxylink{namespace_guzzle_http}{Guzzle\+Http}\textbackslash{}Ring\textbackslash{}Client}.
\item Removed the concept of "{}parallel adapters"{}. Sending requests serially or concurrently is now handled using a single adapter.
\item Moved {\ttfamily \doxylink{namespace_guzzle_http}{Guzzle\+Http}\textbackslash{}Adapter\textbackslash{}Transaction} to {\ttfamily \doxylink{namespace_guzzle_http}{Guzzle\+Http}\textbackslash{}Transaction}.
