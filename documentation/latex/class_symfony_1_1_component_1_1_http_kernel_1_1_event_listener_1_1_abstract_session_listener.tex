\doxysubsection{Detailed Description}
Sets the session onto the request on the "{}kernel.\+request"{} event and saves it on the "{}kernel.\+response"{} event.

In addition, if the session has been started it overrides the Cache-\/\+Control header in such a way that all caching is disabled in that case. If you have a scenario where caching responses with session information in them makes sense, you can disable this behaviour by setting the header Abstract\+Session\+Listener\+::\+NO\+\_\+\+AUTO\+\_\+\+CACHE\+\_\+\+CONTROL\+\_\+\+HEADER on the response.

Returns an array of event names this subscriber wants to listen to.

The array keys are event names and the value can be\+:


\begin{DoxyItemize}
\item The method name to call (priority defaults to 0)
\item An array composed of the method name to call and the priority
\item An array of arrays composed of the method names to call and respective priorities, or 0 if unset
\end{DoxyItemize}
