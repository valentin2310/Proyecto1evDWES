This is mostly a cleanup release offering stronger type guarantees for integrators with the array-\/\texorpdfstring{$>$}{>}object/enum changes, but there is no big new feature for end users.

See \href{UPGRADE.md\#300}{\texttt{ UPGRADE notes}} for details on all breaking changes especially if you are extending/implementing \doxylink{namespace_monolog}{Monolog} classes/interfaces.

Noteworthy BC Breaks\+:


\begin{DoxyItemize}
\item The minimum supported PHP version is now {\ttfamily 8.\+1.\+0}.
\item Log records have been converted from an array to a \href{src/Monolog/LogRecord.php}{\texttt{ {\ttfamily Monolog\textbackslash{}Log\+Record} object}} with public (and mostly readonly) properties. e.\+g. instead of doing `\$record\mbox{[}\textquotesingle{}context'\mbox{]}{\ttfamily use}\$record-\/\texorpdfstring{$>$}{>}context{\ttfamily . In formatters or handlers if you rather need an array to work with you can use}\$record-\/\texorpdfstring{$>$}{>}to\+Array(){\ttfamily  to get back a \doxylink{namespace_monolog}{Monolog} 1/2 style record array. This will contain the enum values instead of enum cases in the}level{\ttfamily and}level\+\_\+name{\ttfamily keys to be more backwards compatible and use simpler data types. -\/}Formatter\+Interface{\ttfamily ,}Handler\+Interface{\ttfamily ,}Processor\+Interface{\ttfamily , etc. changed to contain}Log\+Record \$record{\ttfamily  instead of}array \$record{\ttfamily parameter types. If you want to support multiple \doxylink{namespace_monolog}{Monolog} versions this should be possible by type-\/hinting nothing, or}array\texorpdfstring{$\vert$}{|}\+Log\+Record{\ttfamily if you support PHP 8.\+0+. You can then code against the \$record using \doxylink{namespace_monolog}{Monolog} 2 style as Log\+Record implements Array\+Access for BC. The interfaces do not require a}Log\+Record{\ttfamily return type even where it would be applicable, but if you only support \doxylink{namespace_monolog}{Monolog} 3 in integration code I would recommend you use}Log\+Record{\ttfamily return types wherever fitting to ensure forward compatibility as it may be added in \doxylink{namespace_monolog}{Monolog} 4.}
\item {\ttfamily Log levels are now enums \mbox{[}}\doxylink{namespace_monolog}{Monolog}\textbackslash{}\+Level{\ttfamily \mbox{]}(\doxylink{_level_8php}{src/\+Monolog/\+Level.\+php}) and \mbox{[}}\doxylink{namespace_monolog}{Monolog}\textbackslash{}\+Level\+Name{\ttfamily \mbox{]}(src/\+Monolog/\+Level\+Name.\+php)}
\item {\ttfamily Removed deprecated Swift\+Mailer\+Handler, migrate to Symfony\+Mailer\+Handler instead. -\/}Resettable\+Interface\+::reset()\`{} now requires a void return type.
\item All properties have had types added, which may require you to do so as well if you extended a \doxylink{namespace_monolog}{Monolog} class and declared the same property.
\end{DoxyItemize}
