\item Characters that function as separator between placeholders are now whitelisted to fix routes with normal text around a variable, e.\+g. {\ttfamily /prefix\{var\}suffix}.
\item \mbox{[}BC BREAK\mbox{]} The default requirement of a variable has been changed slightly. Previously it disallowed the previous and the next char around a variable. Now it disallows the slash ({\ttfamily /}) and the next char. Using the previous char added no value and was problematic because the route {\ttfamily /index.\{\+\_\+format\}} would be matched by {\ttfamily /index.ht/ml}.
\item The default requirement now uses possessive quantifiers when possible which improves matching performance by up to 20\% because it prevents backtracking when it\textquotesingle{}s not needed.
\item The Configurable\+Requirements\+Interface can now also be used to disable the requirements check on URL generation completely by calling {\ttfamily set\+Strict\+Requirements(null)}.
