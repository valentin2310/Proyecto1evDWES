\begin{DoxyItemize}
\item Added generic {\ttfamily Common\+Mark\+Exception} marker interface for all exceptions thrown by the library

\begin{DoxyItemize}
\item {\ttfamily Already\+Initialized\+Exception}
\item {\ttfamily Invalid\+Argument\+Exception}
\item {\ttfamily IOException}
\item {\ttfamily Logic\+Exception}
\item {\ttfamily Missing\+Dependency\+Exception}
\item {\ttfamily No\+Matching\+Renderer\+Exception}
\item {\ttfamily Parser\+Logic\+Exception}

\item Added more configuration options to the Heading Permalinks extension (\#939)\+:
\begin{DoxyItemize}
\item {\ttfamily heading\+\_\+permalink/apply\+\_\+id\+\_\+to\+\_\+heading} -\/ When {\ttfamily true}, the {\ttfamily id} attribute will be applied to the heading element itself instead of the {\ttfamily \texorpdfstring{$<$}{<}a\texorpdfstring{$>$}{>}} tag
\item {\ttfamily heading\+\_\+permalink/heading\+\_\+class} -\/ class to apply to the heading element
\item {\ttfamily heading\+\_\+permalink/insert} -\/ now accepts {\ttfamily none} to prevent the creation of the {\ttfamily \texorpdfstring{$<$}{<}a\texorpdfstring{$>$}{>}} link
\end{DoxyItemize}
\item Added new {\ttfamily table/alignment\+\_\+attributes} configuration option to control how table cell alignment is rendered (\#959)

\begin{DoxyItemize}
\item Change several thrown exceptions from {\ttfamily Runtime\+Exception} to {\ttfamily Logic\+Exception} (or something extending it), including\+:
\begin{DoxyItemize}
\item {\ttfamily Callback\+Generator}s that fail to set a URL or return an expected value
\item {\ttfamily Markdown\+Parser} when deactivating the last block parser or attempting to get an active block parser when they\textquotesingle{}ve all been closed
\item Adding items to an already-\/initialized {\ttfamily Environment}
\item Rendering a {\ttfamily Node} when no renderer has been registered for it
\end{DoxyItemize}
\item {\ttfamily Heading\+Permalink\+Processor} now throws {\ttfamily Invalid\+Configuration\+Exception} instead of {\ttfamily Runtime\+Exception} when invalid config values are given.
\item {\ttfamily Html\+Element\+::set\+Attribute()} no longer requires the second parameter for boolean attributes
\item Several small micro-\/optimizations
\item Changed Strikethrough to only allow 1 or 2 tildes per the updated GFM spec
