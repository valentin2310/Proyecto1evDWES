\begin{DoxyItemize}
\item \href{https://github.com/sebastianbergmann/phpunit/pull/5505}{\texttt{ \#5505}}\+: Improve the failure description of {\ttfamily String\+Contains}-\/based assertions when the strings are encoded differently
\item \href{https://github.com/sebastianbergmann/phpunit/issues/5515}{\texttt{ \#5515}}\+: The {\ttfamily Test\textbackslash{}Assertion\+Succeeded} and {\ttfamily Test\textbackslash{}Assertion\+Failed} events are always emitted again
\item \href{https://github.com/sebastianbergmann/phpunit/issues/5515}{\texttt{ \#5515}}\+: {\ttfamily -\/-\/log-\/events-\/verbose-\/text} enables the export of non-\/scalar values for the {\ttfamily Test\textbackslash{}Assertion\+Succeeded} and {\ttfamily Test\textbackslash{}Assertion\+Failed} events
\item \href{https://github.com/sebastianbergmann/phpunit/issues/5524}{\texttt{ \#5524}}\+: Do not export data passed to test for \doxylink{namespace_p_h_p_unit}{PHPUnit}\textbackslash{}\+Event\textbackslash{}\+Code\textbackslash{}\+Test\+Method value object by default

\item Exceptions thrown by third-\/party event subscribers are now reported as test runner warnings
\item The name of the top-\/level test suite that is created when a directory or file path is passed as an argument to the test runner is now {\ttfamily CLI Arguments}
\item Simplified the failure description for {\ttfamily assert\+Instance\+Of()} and {\ttfamily assert\+Not\+Instance\+Of()}
\item Simplified the failure description for {\ttfamily assert\+Json()}
\item The {\ttfamily -\/-\/enforce-\/time-\/limit} CLI option and the {\ttfamily enforce\+Time\+Limit} XML configuration file attribute no longer affect the running of PHPT tests
\item The {\ttfamily @code\+Coverage\+Ignore}, {\ttfamily @code\+Coverage\+Ignore\+Start}, and {\ttfamily @code\+Coverage\+Ignore\+End} annotations are no longer deprecated
\end{DoxyItemize}
