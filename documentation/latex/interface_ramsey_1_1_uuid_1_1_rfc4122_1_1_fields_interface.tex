\doxysubsection{Detailed Description}
RFC 4122 defines fields for a specific variant of UUID

The fields of an RFC 4122 variant UUID are\+:


\begin{DoxyItemize}
\item {\bfseries{time\+\_\+low}}\+: The low field of the timestamp, an unsigned 32-\/bit integer
\item {\bfseries{time\+\_\+mid}}\+: The middle field of the timestamp, an unsigned 16-\/bit integer
\item {\bfseries{time\+\_\+hi\+\_\+and\+\_\+version}}\+: The high field of the timestamp multiplexed with the version number, an unsigned 16-\/bit integer
\item {\bfseries{clock\+\_\+seq\+\_\+hi\+\_\+and\+\_\+reserved}}\+: The high field of the clock sequence multiplexed with the variant, an unsigned 8-\/bit integer
\item {\bfseries{clock\+\_\+seq\+\_\+low}}\+: The low field of the clock sequence, an unsigned 8-\/bit integer
\item {\bfseries{node}}\+: The spatially unique node identifier, an unsigned 48-\/bit integer
\end{DoxyItemize}

{\footnotesize\ttfamily get\+Variant (\begin{DoxyParamCaption}{ }\end{DoxyParamCaption})}

Returns the variant

The variant number describes the layout of the UUID. The variant number has the following meaning\+:


\begin{DoxyItemize}
\item 0 -\/ Reserved for NCS backward compatibility
\item 2 -\/ The RFC 4122 variant
\item 6 -\/ Reserved, Microsoft Corporation backward compatibility
\item 7 -\/ Reserved for future definition
\end{DoxyItemize}

For RFC 4122 variant UUIDs, this value should always be the integer {\ttfamily 2}.

{\footnotesize\ttfamily get\+Version (\begin{DoxyParamCaption}{ }\end{DoxyParamCaption})}

Returns the version

The version number describes how the UUID was generated and has the following meaning\+:


\begin{DoxyEnumerate}
\item Gregorian time UUID
\item DCE security UUID
\item Name-\/based UUID hashed with MD5
\item Randomly generated UUID
\item Name-\/based UUID hashed with SHA-\/1
\item Reordered time UUID
\item Unix Epoch time UUID
\end{DoxyEnumerate}

This returns {\ttfamily null} if the UUID is not an RFC 4122 variant, since version is only meaningful for this variant.
