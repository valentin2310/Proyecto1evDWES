\chapter{This is the PHP port of Hamcrest Matchers}
\hypertarget{md_vendor_2hamcrest_2hamcrest-php_2_r_e_a_d_m_e}{}\label{md_vendor_2hamcrest_2hamcrest-php_2_r_e_a_d_m_e}\index{This is the PHP port of Hamcrest Matchers@{This is the PHP port of Hamcrest Matchers}}
\href{https://travis-ci.org/hamcrest/hamcrest-php}{\texttt{ }}

\doxylink{namespace_hamcrest}{Hamcrest} is a matching library originally written for Java, but subsequently ported to many other languages. hamcrest-\/php is the official PHP port of \doxylink{namespace_hamcrest}{Hamcrest} and essentially follows a literal translation of the original Java API for \doxylink{namespace_hamcrest}{Hamcrest}, with a few Exceptions, mostly down to PHP language barriers\+:


\begin{DoxyEnumerate}
\item {\ttfamily instance\+Of(\$the\+Class)} is actually {\ttfamily an\+Instance\+Of(\$the\+Class)}
\item `both(contains\+String(\textquotesingle{}a'))-\/\texorpdfstring{$>$}{>}and(contains\+String(\textquotesingle{}b\textquotesingle{})){\ttfamily  is actually}both(contains\+String(\textquotesingle{}a\textquotesingle{}))-\/\texorpdfstring{$>$}{>}and\+Also(contains\+String(\textquotesingle{}b\textquotesingle{}))\`{}
\item `either(contains\+String(\textquotesingle{}a'))-\/\texorpdfstring{$>$}{>}or(contains\+String(\textquotesingle{}b\textquotesingle{})){\ttfamily  is actually}either(contains\+String(\textquotesingle{}a\textquotesingle{}))-\/\texorpdfstring{$>$}{>}or\+Else(contains\+String(\textquotesingle{}b\textquotesingle{}))\`{}
\item Unless it would be non-\/semantic for a matcher to do so, hamcrest-\/php allows dynamic typing for it\textquotesingle{}s input, in "{}the PHP way"{}. Exception are where semantics surrounding the type itself would suggest otherwise, such as string\+Contains() and greater\+Than().
\item Several official matchers have not been ported because they don\textquotesingle{}t make sense or don\textquotesingle{}t apply in PHP\+:
\begin{DoxyItemize}
\item {\ttfamily type\+Compatible\+With(\$the\+Class)}
\item {\ttfamily event\+From(\$source)}
\item {\ttfamily has\+Property(\$name)} \texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}
\item {\ttfamily same\+Property\+Values\+As(\$obj)} \texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}
\end{DoxyItemize}
\item When most of the collections matchers are finally ported, PHP-\/specific aliases will probably be created due to a difference in naming conventions between Java\textquotesingle{}s Arrays, Collections, Sets and Maps compared with PHP\textquotesingle{}s Arrays.
\end{DoxyEnumerate}
